How many times have you finished a great meeting, only to realize that writing up summaries, checking calendars, sending emails, and logging Jira tickets takes longer than the meeting itself? Today, I want to show you how we can completely automate this process securely.

I have built the Meeting Intelligence Agent. It is a multi-agent system designed for business operations that screens raw transcripts for security and PII, extracts action items and decisions, runs a human-in-the-loop review, and connects to enterprise tools via MCP to automate post-meeting workflows.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the system architecture.
Our flow begins with the raw meeting transcript, which is sent to the Security Checkpoint node. The checkpoint screens the transcript, automatically redacting emails, phone numbers, and credentials, and checking for prompt injection.
If the transcript passes the checkpoint, it is prepared and sent to our first specialized agent, the Transcript Analyzer.
The Analyzer extracts the meeting date, attendees, decisions, and action items, using the MCP Server's calendar lookup tools to verify dates and attendees.
Next, we pause at the Human Review Gate. This is our human-in-the-loop check.
Once a human reviews and approves the data, the Summary & Notification Agent takes over. It uses directory search to find emails for each owner, registers the tasks as tickets via the MCP Server, and archives the final markdown report.

What makes this system robust is the dual layer of security and the Model Context Protocol server. The security checkpoint prevents data leaks and guards against injection. The MCP server connects the LLM safely to local enterprise data: listing calendar events, searching user directories, and creating Jira-style tickets.
